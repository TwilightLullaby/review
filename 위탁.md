ide sata sas scsi
----------------------
## 요약하면 케이블 종류..    
**IDE ( Integrated Drive Electronics )**
> 가장 오래된 규격 데이터를 병렬로 전송한다는 뜻에서 PATA ( Parallel Advanced Technology Attachment ) 라고도 부른다.
> 최신규격 UDMA6 모드에서는 초당 133.3MB 전송속도를 낸다.          
> 현재는 초기 IDE 보다 성능 향상된 E-IDE ( Enhanced IDE ) 규격을 사용하는데, 일반적으로 현재 IDE 라고 부르는 녀석이 E-IDE 이다
* * *
**SATA ( Serial Advanced Technology Attachment )**
> SATA1 에서는 초당 150MB            
> SATA1 에서는 초당 150MB          
> SATA2 에서는 초당 300MB        
> SATA2 에서는 USB 처럼 허브를 이용해 하나의 포트에 여러 개의 하드를 연결할 수 있고, PC 를 끄지 않고 장치를 연결하는 핫플러그도 지원한다.          
> 최근에는 SATA3 도 나왔다. 초당 600MB      
* * *
**SCSI ( Small Computer System Interface )** 
> 서버나 워크스테이션 등에 쓰이는 고속 인터페이스다.      
> 안정성이 높은 것이 최대 장점          
> 가격이 비싼 것은 단점    
> 최신 규격인 울트라 320은 최대 320MB/s 의 속도를 낸다.
> 최근 SAS 가 등장하면서 조금씩 퇴색되고 있다.
* * *
**SAS (Serial Attached SCSI**
> SCSI 를 발전시킨 것으로 마찬가지로 서버나 워크스테이션 등에서 쓰인다.
> 울트라 320 SCSI 보다 좋다.
> 커넥터와 선은 SATA 와 같은 것을 사용한다.
SATA 규격 하드를 SAS 장치에 꽂아 쓸 수는 있지만, 반대로 SAS 하드를 SATA 장치에 꽂아 사용하지는 못한다.

Directory Sevice
--------

디렉터리 서비스는 네트워크 관련 자원 정보를 중앙의 저장소에 통합시켜 놓음으로써 더 이상 사용자가 네트워크상의 정보를 찾기 위해서 해매지 않도록 도와준다. 즉 사용자는 중앙의 저장소를
통해서 원하는 정보에 대한 정보를 자동으로 취득하게 되고 네트워크 자원에 접근할 수 있는것이다. 여기서 네트워크 자원이란 사용자계정,프린터 등의 주변기기를 말한다

Acitive Directory 도메인서비스
--------
컴퓨터,사용자 기타 주변장치에 대한 정보를 네트워크상 저장하면 Active Directory 도메인 서비스는 이러한 정보들을 관리자가 통합하여 관리하게 된다.
이렇게하면 사용자는 네트워크에 편리하게 자원들을 공유할수있으며 공동으로 작업도 가능해진다 Active Directory 도메인 서비스를 사용하기 위해서는 기본적으로
dns서버를 설치해야한다

DNS=DNS
-------
DNS는 도메인네임서버를 일컫는다. 인터넷은 서버들을 유일하게 구분할 수 있는 IP주소를 기본체계로 이용하는데 숫자로 이루어진 조합이라 인간이 기억하기에는 무리가 따른다. 따라서 DNS를 이용해 IP주소를 인간이 기억하기 편한 언어체계로 변환하는 작업이 필요한데 이 역할을 DNS가 하는 것이다.
그냥 요약하면 ip를 쓰기힘들니까 아이피를 언어올 바꾼거다 
도메인->www.naver.com 이런것들이 인간의언어로 되어있는이유는 dns서비스때문에 있는거다
그리고 www.naver.com 이거는 사실 www.naver.com. 뒤에 점이 하나 더 붙는다 이걸 루트동행이라한다 그래서 도메인 읽을떄는 뒤부터 읽어야한다 .(루트동행)(이건안보인다)->.com->naver->www. 이렇게 읽어야한다

순서
처음에는 .(루트도메인)시작하고
com=컴퍼니 학교는 ac=아카데미 청와대는 go = 거버먼트 mil=밀리터리 뒤에 끝나는 이런게 도메인의 첫번째 레벨 (1레벨)
naver, google 2번째 레벨이다 (2레벨)
www 3번째 레벨 (3레벨)

도메인
도메인은(Domain)은 Active Directory의 가장 기본이 되는 단위로써, 서울본사,부산지사 등이
각각 하나의 도메인이라고 보면 된다, 즉 도메인은 관리를 하기 위한 하나의 큰 단위의 범위를 표현하며,관리를 위해서 지역적인 범위로 구분할수있다
즉 도메인은 하나의 범위다

트리와 포리스트
---------
트리(Tree)는 도메인의 집함으로 보면된다. 부모 도메인인 서울지사와 자식 도메인인 부사지사와, 일본사무실이 계층적 구조로 되어 있는데 이를 하나로     
묶은 한빛 주식회사를 "트리 구조" 또는 그냥 "트리"라고 부른다 그래서 트리는 물리적으로 존재하다기보다는 개념적인 것으로 보면 된다. 한빛 주식회사는 단일 트리로 구성되어있는데   
만약 디스 주식회사라는 다른 회사와 전략적 제휴를 맺게 된다면 두 회사는 서로 관계를 맺을 수가 있을 것이다 , 즉 두 개 이상의트리로 Active Directory가 구성되는데 이를 포리스트라고한다

> 도메인<트리<포리스트          


사이트
----------
도메인이 논리적인 범주라면, 사이트는(Site)는 물리적인 범주에 가깝다   
서울 본사와 일본 사무실은 Hambit.com으로 같은 도메인으로 묶여 있지만, 실제로는 거리상 상당히 떨어져 있다.    
그러므로 서울 본사와 일본 사무실은 같은 도메인이지만 별도의 사이트로 구성된다     


트러스트
---------
도메인 또는 포리스트 사이에 신뢰할 지 여부에 대한 관계를 나타내는 의미로 사용된다. 우선 같은
포리스트 안의 도메인 사이에는 상호 양방향전이 트러스틀(Trust)를 갖는다 이 외에도 한쪽 방향의 
신뢰관계만 맺은 경우를 '단반향 전이 트러스터'라 부른다



조직 구성 단위
------------
조직 구성 단위(organizational unit, OU)는 한 도메인 안에서 세부적인 단위로 나누는 것을 말한다.
서울 본사를 관리부,회계부,기술부 등의 부서로 나눈다면 이러한 부서를 '조직 구성 단위'라 함

글로벌 카탈로그
----------
Active Directory 트러스트 내의 도메인들에 포함된 개체에 대한 정보를 수집하여 저장되는 통합 저장소를
글로벌 카탈로그(Global Cartlog)라고 말한다. 예를 들어 사용자의 경우에는 사용자의 이름,전체 이름,로그인 아이디와 비밀번호 등의 정보가 글로벌 카탈로그에 저장된다.

액티브 디렉터리 사용자 계정과 조직 구성 단위
===
사용자 계정은 Active Directory와 관련이 없이 로컬 컴퓨터만 접근할 수 있는 '로컬 사용자 계정'과
Active Directory 도메인에 접근할 수 있는 '도메인 사용자 계정'으로 나뉜다

'로컬 사용자 계정'은 Active Directory를 구성하기 이전에 사용하던 계정으로 이후로는 사용하지 않는다.
주로 언급하는 사용자 계정은 Active Directory를 그상한 이후의 사용자 계정인 도메인 사용자 계정이 된다
즉 로컬 컴퓨터뿐 아니라 Active Directory 도메인 전체에 대한 로그온 할 수 있는 사용자를 생성하고 관리할 것  이다

도메인 사용자 계정의 이름은 4가지 정도로 표현할 수 있는데, 만약 yonsai.com 도메인 (netbios)는
yonsai에 사용자 이름을 thisuser로 생성할 경우에 다음과 같이 나타낼 수 있다.

+ 기본적인 도메인 로그온 이름:YONSAI\thisuser 이거 시험에나옴 중요 ★
+ UPN : thisUser@yonsai.com
+ Dishguished name : CN=thisuser, OU=조직 구성단위이름 DC=yonsai, DC=com DC 2개는 도메인
+ Relative Dishguished name : CN:thisUser(조직 구성단위 안에서만 사용 가능)

+Dishguished name에서 CN은 Common Name, OU는 Organizational Unit, DC는
Domain Compoent를 의미한다 Organizational Unit(조직구성단위)

조직 구성 단위(organization Unit)
------
OU는 사용자,그룹 컴퓨터를 포함할 수 있는 Active Directory 컨테이너로, 쉽게 비교하자면 회사의 관리부,회계부,기술부 등의 부서로 정도로 생각하면된다. ou는=사용자,그룹     그냥 ou=풀더라고 생각하셈
![image](https://user-images.githubusercontent.com/76859458/112263166-31a69480-8cb2-11eb-9ac4-50e3c8f26666.png)
OU 특징   
Active Directory의 다른 종류의 개체를 보관할 수 있다.
Domain의 기능을 일부 가지고 있는 개체를 담을 수 있는 하위구조가 가능한 컨테이너 (ou안에 ou가능)
별도의 제어 위임/그룹정책을 줄 수 있다.
회사가 자사를 구성할 경우 도메인 혹은 OU로 구성하고자 할 경우 지역적으로 멀거나 업무적으로 독립된 성향
,독립된 계정 정책을 사용할 경우엔 별도의 자식 도메인을 사용한다.
하지만 일반적인 경우엔 OU를 생성하여 관리의 효율성을 높이는 것이 좋다

도메인 사용자 계정과 OU 생성 방법 
1)사용자 및 컴퓨터에서 도메인 우클릭
2)새로 만들기
3)조직 구성 단위
컴4)도메인 내에 생성 되어진 OU에 "새 개체"(사용자 계정)을 생성한다

액티브 디렉터리 개체 생성 및 삭제 방법
1.eduteam 이름으로 OU를 생성->dsadd ou "ou=eduteam,dc=yonsai,dc=com"

3.정동현 계정을 생성을 User 컨테이너로 옮기기->dsmove " "cn=정동현,dc=yonsai,dc=com" -newparent "
4.정동현 사용자 eduteam OU로 이동시키기 ->dsmove "cn=정동현,dc=yonsai,dc=com" -newparent "OU=eduteam,DC=yonsai,DC=com"
5.(여기서 pwd는 패스워드다 upn는 아이디)

액티브 디렉터리 그룹
===
+ 그룹은 사용자 또는 컴퓨터의 집합으로 보면 된다. 또한, 그룹은 다른 그룹을 포함할 수도 있다
+ 그룹의 가장 큰 목적은 관리자 권한 부여에 있따.
+ 액티브 디렉터리에서 다루는 그룹은 크게 글로벌 그룹, 도메인 로컬 그룹, 유니버셜 그룹등 세가지로 분류된다


글로벌 그룹
----
+ 글로벌 그룹은 "Global"이라는 이름이 의미하듯이, 모든 도메인의 위치한 자원(공유 풀더,프린터 등)에 권한을 할당할 수 있는 그룹이다
+ 주의할 점은 그룹에 속할 구성원은 꼭 글로벌 그룹에 생성한 도메인의 구성원만 포함될 수 있다.
+ 밑에 그림 hanbit.com 도메인 사용자는 당연히 자신의 도메인에 생성된 글로벌 그룹에 포함될 수 없다
+ hanbit.com도메인의 도메인의 글로벌 그룹은 자신의 도메인의 자원뿐만아니라 다른 도메인의 자원에도 접근 할수있다(물론 접근권한줘야한다)
+ 글로벌 그룹에 포함될 구성원은 다음과 같다 1.같은 도메인에 있는 사용자 계정 또는 컴퓨터계정 2.같은 도메인의 다른 글로벌 그룹

로컬 그룹 
-----

+ 도메인 로컬 그룹에 소속할 수 있는 구성원은 다음과 같다 1.모드 도메인의 사용자 계정 또는 컴퓨터 계정 2.모든 도메인의 글로벌 그룹 및 유니버셜 그룹 3.같은 도메인의 다른 도메인 로컬그룹

windows sever에서 제공하는 기본 그룹(시험에서 나옴 중요★)
======== 
windows sever에서는 사용자가 생성한 그룹 외에도 windows server 자체에 포함된 기본그룹(default groups)이 있다 기본 그룹에는 windows server를 설치하면 들어 있는
기본 로컬 그룹 (Default local gorps)과 active diectory 도메인을 만들면 자동으로 만들어지는 기본 도메인 그룹(default domain groups)이 있다
또 기본 도메인 로컬 그룹은 (default domain local groups), 기본 글로벌 그룹(default global groups)기본 유니버설 그룹(default universal groups)으로 구분할 수 있다.

그룹 정책의 기본 개념과 GPO
---
그룹 정책은 관리자가 액티브 디렉터리 내의 컴퓨터나 사용자에게 사용 가능한 프로그램을 지정하거나 제한할 수 있으며 로그온 시 보여지는 바탕하면을 지정할 수 있고
시작메뉴의 사용옵션 usb나 cd/dvd의 사용제한 등을 구성할 수 있다
그룹 정책을 적용하면 잘못된 사용자의 시스템 구성 변경이나 네트워크에 바이러스 침수
등에 대한 예방이 가능하며 결과적으로는 전반적인 액티브 디렉터리 시스템의 생산성 향상시킨다
그룹 정책을 생성한 후에 그 그룹 정책을 묶은 개체를 그룹 정책 개체 gpo라고 부르는데 gpo는 컴퓨터에 대해서 설정할 수 있고 사용자에 대해서 설정할수 있다
예를들어 컴퓨터에 대해서 설정하는 경우에는 교육 장의 모든 컴퓨터가 부팅시에 같은 환경으로 부팅되도록 할수 있다 
또 사용자가 설정하는경우에는 특정응용프로그램을 사용하거나 사용하지 못하도록 설정하거나 사용자 로그온 또는 로그아웃시에 미리 설정한 스크립트가 실행되도록 할수있다
컴퓨터 설정한 그룹 정책과 사용자 설정한 그룹 정책이 충돌되는 경우에는 우선 순위는 "컴퓨터"에 설정한 그룹 정책이 더 높다
GPO의 저장은 도메인 단위에 저장되는데 글로벌 카탈로그에 해당 정보가 저장된다고 생각하면 된다
### 요약 : 컴퓨터랑 사용자 충돌하면 우선순위는 컴퓨터 GPO=그룹정책 모음  GPO는 Gc경 
그룹 정책의 종류
+ 로컬 GPO
+ 사이트GPO
+ 도메인 GPO
+ OU GPO

GPO가 적용되는 순서는 로컬 GPO부터 적용된다. 그러므로 제일 마지막에 적용되는 OU GPO가 가장 우선순위가 높다고 보면 된다 또한 GPO를 적용하기 위한 가장 작은 단위도
OU가 되며 일반적으로 부서 등과 같은 작은 조직 단위를 OU로 구분하며 GPO를 적용시킨다
사이트 GPO는 지역적으로 묶인 곳에 그룹 정책을 적용하기 위해 주로 사용되는데 자주 사용하되지않는다


그룹 정책의 상속과 가능한 작업
------
그룹 정책은 상속이 되는데, 일반적으로 부모 컨테이너에서 자식 컨테이너로 상속이 된다
예로 yonsai.com도메인에 기술부 OU가 있고 기술부 OU안에는 기술1팀 OU와 기술 2팀 OU가 있다고 가정하면
yonsai.com 도메인 -> 기술부OU-> 기술 1팀 OU의 순서로 그루 정책이 상속 된다 또한 무조건 상속받지 않고 필요할 경우에는 상속을 재정의 하거나 상속을 차단할수있다
예를들어 yonsai.com 도메인에 그룹 정책을 기술1팀 OU에는 적용되지 않도록 상속을 차단하는 것도 가능하다 반대로 상속 차단을 하지 못하도록 강제 상속하는 것도 가능하다
예를들어 회사(도메인)의 정책을 부서(OU)가 거부하지 못하도록 하는것과 비슷한 개념이다 그러므로 강제 상속은 가장 높은 우선 순위를 갖는다.
그룹정책을 사용해서 관리자는 다음 작업을 할 수 있다.
+ 보안 설정 :보안 강화를 위한 사용자의 암호 및 계정 잠금 등에 대헤서 도메인의 모든 사용자에게 강제로 적용할 수 있다
+ 스크립트 지정 : 사용자의 로그온/로그아웃 시에 또는 컴퓨터의 부팅과 종료시에 자동으로 실행될 작업을 스크립트에 지정할 수 있다
+ 풀더 리디랙션(Folder Redirection) : 사용자가 도메인 내의 어느 컴퓨터에서 로그온하더라도 자신의 문서 등에 대한 풀더가 동일한 환경으로 제공되도록 할 수 있다.
+ 소프트웨어 설정 : 사용자가 사용할 소프트웨어에 대해서 설치, 삭제,업데이트를 제어할 수 있다.

참고로 windows server에서 제공하는 그룹 정책의 종류는 수천개가 넘으며, 운영체제가 업데이트되면 그 개수도 더 늘어난다, 이외에도 좀 더 고려할 세부적인 사항이 있지만 그룹 정책에 대한 개념
은 실습을 통해서 이해하자

풀더 리디렉션(Folder Redirection)은 도메인의 사용자가 도메인 내의 어떤 컴퓨터에서 접속하든지 자신이 사용하던 풀더가 그대로 보이도록 하는 것이다

BitLocker 드라이브 암호화
----------
BitLocker 하드디스크 자체를 암호화하기 위한 기술로써, 주 목적은 하드디스크를 통째로 도난당해도 
암호화 키가 없이는 잃지 못하게 하는 것이다
지금까지 BitLocker 기능을 사용하지 않은 하드디스크는, 하드디스크를 고의 또는 실수로 다른
컴퓨터에 연결하면 그 데이터가 그대로 잘 보인다. 즉 BitLocker로 잠글을 하지 않은 일반적인
하드 디스크는 다음과 같이 적용된다

사용자 -----> Windows/하드디스크/사용자파일 <----- 해커(데이터가 잘 보임)

그러나 BitLocker를 적용하여 하드디스크 자체를 접근하려면 강력한 하드웨어적인 암호가 
걸려 있기 때문에 하드디스크가 분실되더라도 함부로 데이터에 접근할 수가 없다.


사용자 -----> Windows/하드디스크/사용자파일/BitLocker <----- 해커(데이터가 잘 보임)

BitLocker를 사용하기 위해서는 일반적으로 TRM(Trusted Platform Module)이라고 부르는
마이크로칩이 메인보드에 내장되어 있어야 한다. 또한 BIOS에서 TPM기능을 지원해줘야
한다. 이러한 하드웨어적 기능은 거의 조작이 불가능한 강력한 암호화를 제공해준다.
* TPM이 없어도 소프트웨어 방식으로 BitLocker 암호화가 가능하다. 우리는 가상머신을 사
용 중이어서 TPM을 사용하지 못하므로, 소프트웨어 방식을 사용한다, 소프트웨어 방식은
암호를 관리하는 방식이므로 하드웨어 방식보다는 약한 암호화라고 생각하면 된다.

BitLocker는 하드디스크를 통째로 잠그는 기능이다. 풀더/파일 단위로 암호화하는 방식인
EFS(Elastic File System = 암호화된 파일 시스템)
또한, Windows Sever 2012부터는 BitLocker의 새로운 기능이 여러개 추가되었다. 특히 이
전에는 드라이브 전체가 암호화되어서 대용량 디스크의 경우 상당히 오랜 시간이 걸렸으나
지금은 사용한 디스크 공간만 암호화시킬 수 도 있어서 암호화 시간을 상당히 줄일 수 있다

암호화
===========
암호화(暗號化) 또는 엔크립션(encryption)은 특별한 지식을 소유한 사람들을 제외하고는 누구든지 읽어볼 수 없도록 알고리즘을 이용하여 정보(평문을 가리킴)를 전달하는 과정이다. 이러한 과정을 통해 암호화된 정보(암호문)를 낳는다. 이에 역행하는 과정을 해독 또는 디크립션(decryption)이라고 하며 이로써 암호화된 정보를 다시 읽을 수 있다.
암호화는 군사와 정부의 은밀한 대화를 위하여 오랫동안 이용되어 왔다. 암호화는 수많은 종류의 시민 체계의 정보를 보호하는 데 흔히 쓰이고 있다. 이를테면 CSI(Computer Security Institute)의 2007년 보고에 따르면 71%의 기업이 일부 데이터 전달을 위하여 암호화를 이용하며 53%가 일부 데이터 저장을 위해 암호화를 이용하는 것으로 조사되었다

알고리즘

감사 정책
======
감사(auditing)란 사용자의 작업이나 시스템의 활동을 추적하고 감시하는 것을 말한다. 감사
정책은 관리자에게 통지할 이벤트를 지정하여, 그 이벤트가 발생하면 그 내용을 컴퓨터에
에 기록하게 된다 관리자는 이러한 이벤트를 모니터링하고 문제점을 파악하여 시스템에 문제가 
발생할지를 미리 차단하는데 활용할 수 있다

* 앞에서 배운 BitLocker가 자물쇠 개념이였다면 이번에 배우는 CCTV 개념으로 생각하면
된다 CCTV를 설치했다고 도난사고가 방지되는 것은 아니지만 , 도난사고 발생 시에 어떤
일이 일어났는지 사후에 확인이 가능한 것과 비슷하다

감사 이벤트의 분류
+ 계정 로그온 이벤트(Active Directory 환경 로그온)
+ 계정 관리 이벤트
+ 디렉터리 서비스 이벤트
+ 개체 엑세스 이벤트 
+ 정착 변경 이벤트
+ 권한사용 이벤트
+ 프로세스 추적 이벤트
각 분류는 여러 개의 하위 이벤트 목록을 가지고 있는데 예를 들어 특정 파일이나 풀더
를 접근한 사용자 꼐정이나 사용자가 컴퓨터에 로그온 또는 로그오프한 시간, 정책을 변경
한 시간등을 기록으로 남길 수 있다. 하위 이벤트의 상세한 목록은 나열하지 않고, 실습에서
몇가지를 확인한다 기록된 이벤트는 [이벤트 뷰어]에서 확인할 수 있다.







감사 정책의 장점★
+ 외부 공격을 찾을 수 있다.
+ 외부 공격에 대한 피해를 확인할 수 있다
+ 향후 피해를 예방하는 정책을 수립할 수 있다
+ 시스템이 문제가 될 정도로 성능이 나빠지는 기준을 정할 수 있다

하지만 감사에 대한 이벤트를 너무 많이 구성할 경우에 그 기록되는 양이 많아져 시스템의
성능이 나빠질 수 있으며, 많은 이벤트 기록 중에 어떤 것이 문제가 있는 것인지 찾기도 어
여려워질 수가 있으므로 꼭 필요한 이벤트에 대해서만 검사가 기록되도록 설정하는 것이 바람직하다

파일 시스템(File system)
서버나 PC용 하드디스크를 위한 Windows의 파일 시스템은 FAT->FAT32->NTFS->ReFS순서로(시험에나옴★)
발전되어 왔다.FAT 및 FAT32방식은 아주 예전의 Windows95/98에서 지원되던 파일 시스템
방식이며 FAT32의 경우에는 현재 USB메모리 등에만 일부 적용되어 사용하고 있다.

가장 일반적인 파일 시스템은 NTFS(New Technology File System)인데, 1993년 WIndows NT 3.1 이 사용되고 있다
한 개 파일 크기는 16TB까지 지원되며 파일 압축, 보안, 쿼터, 볼륨 크기 재활당 등의 특징을 갖는다.
NTFS가 현재도 무리 없이 사용되어 있지만 Windows Sever2012 (또는Windows8)부터는 새로운
파일 시스템인 ReFS(Resilient File System 복원 기능의 파일 시스템)를 지원하기 시작했다
ReFS의 특징을 몇 가지 요약하면 다음과 같다
+ NTFS의 장점과 호환성을 최대한 유지한다
+ 데이터 오류를 자동으로 확인하고 수정하는 기능을 갖는다
+ 데이터가 손상된 경우에도 실시간으로 최대한 데이터를 복구한다. 즉 온라인상에서 작업이 진행된다
+ 대용량 볼륨 및 파일의 크기를 지원한다.

결국 ReFS의 가장 큰 특징은 저장된 데이터의 대한 안정성이 상당히 높아지며 성능이나 최대
지원 디스크/파일 크기가 NTFS에 비해서 개선된 것으로 불 시 있다. 단. Windows Sever 2019까지는
ReFS로 포맷된 디스크는 부팅이 지원되지 않기에 운영체제를 설치할 수 없다


그외에도 exFAT라는 파일 시스템이 있는데 주로 플래시 메모리(USB메모리 SD카드 SSD)에
사용하기 위한 파일 시스템으로 개발되었으며 windows sever 2008이후 부터 지원된다
FAT32의 경우 파일 한개의 크기가 4GB로 제한되지만 exFAT의 경우에는 파일 크기의 제한이
없다. 또한 NTFS로 포맷한 플래시 메모리는 스마트폰 등의 기기에서 인식하지 못하는 경우가
있으니 exFAT로 포맷한 경우에는 FAT32와 같이 일반 스마트기기에서 잘 인식한다

디스크에 용량이 쪼금있는이유는 파일시스템이 차지하고 있기때문이다

RAID
==
파티션은 하나의 물리적인 디스크를 여러개의 파티션으로 나누는 것이고 반대로 여러개의 디스크를
하나로 묶어서 사용하는 방식을 RAID라고 한다
약자로 RAID(Redundant Array of Independent Disk)의, 원래 목적은 저렴한 저용량의 디스크 여러개를 하나의 비싼 대용량의 디스크로 사용하고자 하는 것이였다.


예를 들어 1TB 하드디스크 10개를 가지고, 마치 10TB 하드디스크 1개처럼 사용하고자 하는 것이다
그렇게 디스크를 구성함으로써 여러 가지 좋은 점을 얻을 수 있다 RAID는 구성 방식에
따라서 입출력 성능의 향상이나 디스크 ★결함 허용(Fault Tolerance : 여러개의 디스크 중에서 하나 또는 일부가 고장나도 데이터의 안전을 보장)
의 특징을 갖게 할 수 있다.
RAID는 크게 하드웨어 RAID와 소프트웨어 RAID 두가지로 나눌 수 있다.
하드웨어 RAID는 하드웨어 장치로 RAID를 구성하는 것으로, 별도의 장치에 여러 개의 디스크를 장착하여 하나의 디스크처럼 사용한다
하드웨어 RAID는 안전적이고 효율적이지만 가격이 꽤 비싸다, 하드웨어 RAID는 각 제조사마다 지원하는 방식이 다를 수 있으므로 여기에서는 사용하지 않는다.

소프트웨어 RAID는 컴퓨터에 장착된 여러 개의 디스크를 운영체제 레벨에서 RAID로 구성하는
기능을 말한다 Windows 서버,리눅스 서버,유닉스 서버에서는 다양한 소프트웨어 RAID를 지원한다. 하드웨어 RAID보다는 안정적인 면이 떨어지지만, 디스크만
추가한다면 저렴한 비용으로 빠르고 안정적인 시스템을 구성할 수 있다는 장점이 있다.
* 요즘 제작된 PC나 노트북용 메인보드도 자체적으로 하드웨어 RAID 기능을 가지고 있는 것이 많이 있다. 
고가의 서버용 하드웨어 RAID보다는 기능이 떨어지지만. 개인용 pc에서 하드웨어 RAID를 사용하고자 할 경우에는 별 무리가 없다

참고로 RAID는 1987년 버클리 대학의 연구원들이 'A case for Redindant Array folnexpensive disk'라는 논문에서 RAID 0~5를 정의 한 것이 시초였다.
추가로 1989년에 RAID 6가 정의되었다

단순 볼륨
-----
단순 볼륨은 디스크 1개로 1개의 볼륨을 만든다, 예를 들면 ABCDEF라는 데이터를 저장하면 그냥
순차적으로 해당 디스크에 데이터가 저장된다

스팬 볼륨
----
스팬 볼륨은 2개 이상의 디스크로 한 개의 볼륨을 만든 것이다. 디스크별로 용량이 1TB라면,스펜 볼륨의 용량은 2TB가 된다 즉, 공간 효율이 100%가 되는 것이며
이는 구성된 용량의 총합을 전부 사용한다는 의미도 된다. 저장되는 방식은 그림에 표현된 것처럼 첫 번째 디스크 데이터가 모두 꽉 찬 후에, 두번째 디스크를 사용한다
예에서 ABCDEF를 저장할 때 첫번째 디스크에 ABCDE를 저장하고 디스크가 꽉 차면 두번째 디스크에 F를 저장하는 방식이다.
### 하나 채우고 다음껄로 넘어가는것 일부분 복구가능

스트라이프 볼륨(RAID0)
---
스트라이프 볼륨은 2개 이상의 디스크로 한 개의 볼륨을 만든 것으로 볼륨과 결과적으로 동일하지만 내부적으로 저장되는 방식에는 차이가 있다
ABCDEF를 저장할때 A는 첫 번째, B는 두 번째 저장하는 방식이다, 이렇게 저장하면 A와 B를 동시에 저장하기 때문에 디스크의 입출력 속도가 꽤 향상된다
한 글자당 1초의 저장시간이 걸린다고 가정하면, 단순 볼륨과 스팬 볼륨은 ABCDEF를 저장하는 6초의 시간이 필요하지만 스프라이트 볼륨은 동시에 저장되는 방식이므로
3초면 저장된다, 그래서 스트라이프 볼륨(RAID0)은 입출력 성능이 가장 뛰어난 방식으로 사용된다

스트라이프 볼륨은 성능은 우수하지만, 만약 두개의 디스크 중 하나라도 고장 난다면 모든 데이터를 잃어버리게 된다. 또한 스트라이프 볼륨을 구성했다면 3개 이상으로도 구성이 가능하다. 만약 10개의 디스크로 스트아리프 볼륨을 구성했다면 입출력 속도는 더욱 좋아지겠지만, 10개 이상의 디스크 중에 1개라도 고장나면 모든 데이터응ㄹ 잃어버리는 위험성이 생긴다. 그래서 이 볼륨방식에 데이터를 저장하는 경우에는 데이터가 손실되더라도 큰 문제가 되지 않는 데이터를 저장하는 것이 좋다.
### 스트라이프는 위에 방식의 반대 하나 채우고 하나채우고 복구불가능 



미러볼륨(RAID2)
------
미러볼륨은 용어에서도 알 수 있듯이 거울처럼 똑같은 디스크를 구성하는 것이다.
그래서 디스크 하나에 문제가 생기더라도 데이터는 잘 보존된다. 실무에서 중요한 데이터를 저장할 때 많이 사용되는 방식이다.
### 디스크 두개에 둘다 때려박는거 근데 용량 비효율적임 ㄹㅇ 미러링

RAID5
----
RAID5는 미러볼륨처럼 데이터의 안정성이 어느 정도 보장되면서 공간 효율성도 좋은 방식이다.
RAID5는 최소한 3개 이상의 하드디스크가 있어야만 구성이 가능하며 실무에서 사용할 때는 대부분 5개 이상의 하드디스크로 구성한다. 
구성원리는 데이터의 저장 시에 패리티(Parity)를 이용함으로써, 디스크에 문제가 발생 시 원래의 데이터를 예측하는 방식이다.
RAID5의 장점은 어느 정도의 결함은 허용하며, 또한 저장 공간의 효율도 좋다는 것이다.
각 디스크별로 1TB라고 가정한다면, 총 사용할 수 있는 공간의 2TB이다 즉 , 전체 용량의 66.6를 사용할 수 있다
만약 RAID5를 10개의 디스크로 구성했다면, 전체 10TB 중 1개의 패리티로 사용하는 1TB를 제외하고 나머지 9TB를 사용할 수 있으므로 전체 용량의 90%를 사용하게 된다.
여기서 디스크의 개수를 N개라고 하면, N-1만큼의 공간을 사용할 수 있다는 계산을 할 수 있다.

장애 조치 클러스터
====
RAID 1(Mirroring), RAID 및 RAID 1+0 등은 문제가 발생할 때도 시스템이 계속 가동된다
그런데, 디스크가아닌 cpu,메인보드,RAM,네트워크 카드등의 하드웨어 장치가 고장났다면 어떻게 될까?
당연히 시스템이 중단될 수 밖에 없을 것이다. 중요한 서비스를 제공하는 회사라면
이러한 시스템 중단이 회사에 치명적인 결과를 가져올 것은 당연한 일이다. 예를들어
인터넷 뱅킹의 마비, 공항 예약 중단, 인터넷 쇼핑을 판매중지 등등 그러한 에는 많이 찾아볼수있다


장애조치 클러스터는 다양한 하드웨어가 고장나도 서비스가 중단없이 제공되는 기능을 제공한다
실무에서 서비스가 중단되면 안되는 중요한 환경이라면 장애 조치 클러스트 환경을 적극 고려해야 한다.

저장소 공간
==
Windows Sever 2012부터 지원되며 Windows Sever 2019에서는 더욱 기능과 안정성이 높아진 저장소 공간(Srorage Space)에 대해서 알아보자
![image](https://user-images.githubusercontent.com/76859458/114115712-4de73a00-991e-11eb-8910-6a8763f63882.png)

저장소 공간의 개념
저장소 공간은 개념은 여러 개의 물리적인 디스크를 하나로 합쳐서, 사용자에게 새로운 논리적인
디스크로 제공하는 기능을 한다. 이렇게 구성하면 예를 들어 사용자는 10GB의 논리적인 디스크를
사용하다가 필요할 경우에 20GB로 재설정해서 사용할 수가 있다. 즉, 간단한 작업만으로도 공간 확장이 즉시 가능하다
이렇게 사용자 필요로 하는 공간을 제공하는 장소를 저장소(Storage Pool)이라고 부른다

* 저장소 공간은 앞에서 실습한 [디스크 관리]의 RAID 구성을 대신할 수도 있다. 또한, 저장소 공간은 Winddows 8/8.1/10에서도 사용할 수 있다

저장소 공간의 구성과 특징
저장소 공간의 구현은 미러/패리티/단순 3가지로 구성할 수 있는데 각각은 다음과 같은 장단점이 있다.
다음 구성은 결국 RAID에서 설명과 비슷하다.

* 미러:RAID1 개념으로 양방향 및 3방향 미러를 지원한다. 양방향은 최소 개, 3방향은 최소 5개의 디스크가 필요하며 50% 또는 33% 미만의 공간 효율을 갖는다. 
하지만 데이터의 안전성은 뛰어나다.
* 패리티:RAID5 개념으로 패리티 비트가 사용된다. 안정성이 어느 정도 올라가지만 용량은 약간 감소된다. 최소 3개 이상의 디스크가 필요하다.
* 단순(복원 없음):RAID0의 개념으로 100% 공간 효율과 뛰어난 성능을 보이지만 안정성은 보장되지 않는다. 1개 이상의 디스크로 구성이 가능하다.
그 외에도 저장소 공간및 몇 가지 특징은 다음과 같다
* 교체 전용 디스크의 예약:핫 스패어(Hot spare)개념의 디스크를 미리 준비해서 미러나 패리티로 구성된 저장소 폴의 디스크에 문제가 발생했을때
신속하게 애용할 수 있다.
* 장애 조치(failover) 클러스터와 완벽하게 통합된다
* 저장소 계층의 분리 : Windows Sever 2012 R2부터는 자주 사용되는 파일을 SSD로 자주 사용되지 않는 파일을 HDD로 분리해서 저장소 계층을 구성할 수 있다.
전반적인 성능 향상이 가능해졌다. 물론 이러한 구성은 Winodows Sever가 내부적으로 자동으로 수행한다.
* 나중 쓰기 캐시 : Widows Sever 2012 R2부터는 전달된 데이터를 SSD로 구성된 저장소 폴에 먼저 저장한 후에 나중에 HDD에 저장되는 방식을 사용할 수 있다. 빠른
데이터 저장과 성능이 보장된다
* 저장소 공간 다이렉트(Storage Spaces Direct):로컬 연결 드라이브가 있는 서버 컴퓨터에서 기존의 SAN이나 NAS에 비해 훨씬 저렴한 비용의 고가용성
소프트웨어 저장소를 제공한다. 특히 SATA SSD 및 NVMe 디스크 장치의 활용성을 높일 수 있다.
* 저장소 복제본(Storage repiica):저장소를 두 장소에 동일하게 복제함으로써, 장애 조치 클러스터를 확장하고 재해 복구를 대비할 수 있다.
저장소 복제본은 동기 복제와 비동기 복제 모두 가능하다.

이 외에도 Windows Sever 2016에서는 저장소 품질 서비스 데이터,중복 제거, 작업 풀더 기능이 추가 되었다.

NAS
-----
Network Attached Storage는 네트워크상에 저장소를 말한다. 일반적으로 파일 서버(저장소)
사용되는 NAS 장치는 두 개 이상의 디스크와 기가 비트gigabit 네트워크 카드를 자체적으로 가지고 있다
NAS 장치는 일반적인 TCP/IP네트워크를 이용하기 때문에, 별도의
네트워크 확장 작업이 필요없이 기존에 사용하면 네트워크 환경에 연결만 하면 추가 된다
![image](https://user-images.githubusercontent.com/76859458/114131022-b8a76e00-993c-11eb-9aab-e6d58364b626.png)
위 그림에서는 기존의 서버 2대아 pc 3대를 운영하는 환경에 TCP/IP를 사용하는 NAS 장치를 추가했다.
특별히 변경한 것은 없으며, 네트워크의 NAS 장치만 추가하면 된다. 4대의
컴퓨터가 NAS장치에 공유된 공간을 공통으로 사용하면 된다.
최근의 NAS 장비는 그 가격이 고가에서 저가까지 여러 회사에서 많은 모델이 출시되고 있으며
, 저가 모델의 경우 가정용으로도 사용될 정도로 보편화 되엇다. 간편한 구성과 저렴한 비용으로 네트워크
스토리지 환경을 구성할 수 있는 장점이 있다. 단점으로는 TOP/IP의 속도 이상으로 저장 속도가 나올 수 없기 때문에 대용량의 저장 및
추출에서 만족할 만한 성능이 나오지 않을 수 있다.
주로 소규모 네트워크 환경에서 구성하는 것이 적절하다. 또한 그림에는 표현되지 않았지만
성능이 좋은 스위치 허브와 같은 장치를 사용하는 것이 NAS의 성능을 위해서 바람직하다.
NAS장치의 경우 RAID1이나 RAID5와 같은 내결함성이 있는 디스크환경으로 구성하는 것이 바람직하다.
* NAS 장비에 RAID 기능을 가지고 있는 것도 많이 있으며, 반대로 RAID 장비가 NAS의 기능을
가지고 있는 장치도 있다. 또한, NAS장비에는 기존 외장 하드디스크와 달리 별도의 전용 운영체제가 작동하고 있다.
* NAS와 SAN은 별도의 하드웨어 장치가 있어야 실습이 가능하기에 실습은 진행하지 못하지만.NAS는 NAS4free라는 소프트웨어를 이용해서 구성해본다

SAN
---
STORAGE AREA NETWORK는 공통으로 사용되는 저장소를 중앙에서 관리함으로써 각각의 컴퓨터에 저장소를 가지고 있을 때보다
여유 공간의 활용도가 높다. 궁극적인 목적은 NAS와 비슷해 보이지만 NAS가 중소규모에서 주로 사용된다면 SAN은 대규모 이상의 환경에
서 주로 구성된다. SAN환경은 저장공간은 중앙에 관리하여 각각의 서버에서 스토리지를 소유하고 있을 때보다 활용도가 훨씬 높아지고있다
![image](https://user-images.githubusercontent.com/76859458/114132441-53a14780-993f-11eb-848b-44248cc989ea.png)
기존 방식은 각 서버에서 자체의 스토리지를 보유하고 있으나, 이를 SAN 환경으로 구성 하면
위 그림의 오른쪽과 같이 파이버 채널FIBER CHANEL 스위치와 주로 광케이블로 구성 된다.
따라서 SAN 방식을 구현하려면 하드웨어적인 별도의 환경이 필요하며 새로 구축할 경우 많은 비용이 요구 될수있다
또한 SAN으로 구현하면 여러 개의 스토리지가 하나의 스토리지로 묶이는 효과를 낼 수 있다. 위 그림의 경우
각 스토리지에 1TB의 여유 공간이 있었다면 SAN방식으로 구성하면 서버1과 2에서는 SAN저장소에 2TB의 여유 공간이 있는
것으로 둘다 인식된다

SAN의 장점을 정리하면 다음과 같다
* 중앙 관리를 통한 신뢰도 향상
SAN의 저장소는 대부분 RAID와 같은 결합 허용이 되는 장치로 구성되며,UPS와 같은 전원 공급 장치로 구성하는 것이 일반적이다
* 저장 공간의 이용률 향상
SAN은 저장 공간을 중앙에서 곤리하게 되므로 여러 스토리지에 분산된 여유 공간을 한 곳에 모으는 효과가 있다
* 데이터 추출 성능의 향상
SAN은 RAID와 같은 고성능 하드웨어 뿐 아닐, 광채널과 같은 형태의 네트워크 환경을 구성하므로
통신 속도가 NAS등과 같은 넽워크 스토리지에 비해서 월등히 빠르다.
* 스토리지의 확장 용이
SAN을 한 번 구성한 후에, 필요할 경우에 스토리지의 확장 및 변경이 용이하다.

SAN의 단점은 다음과 같다.
* 고가의 비용 및 복잡도 증가
SAN은 기존의 스토리지가 직접 연결된 상태나, NAS에 비해서 그 비용이 훨씬 비싸다, 또한 초기에
SAN의 구성을 위한 설계가 다른 네트워크 스토리지에 비해서 복잡하다.
