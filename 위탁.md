ide sata sas scsi
----------------------
## 요약하면 케이블 종류..    
**IDE ( Integrated Drive Electronics )**
> 가장 오래된 규격 데이터를 병렬로 전송한다는 뜻에서 PATA ( Parallel Advanced Technology Attachment ) 라고도 부른다.
> 최신규격 UDMA6 모드에서는 초당 133.3MB 전송속도를 낸다.          
> 현재는 초기 IDE 보다 성능 향상된 E-IDE ( Enhanced IDE ) 규격을 사용하는데, 일반적으로 현재 IDE 라고 부르는 녀석이 E-IDE 이다
* * *
**SATA ( Serial Advanced Technology Attachment )**
> SATA1 에서는 초당 150MB            
> SATA1 에서는 초당 150MB          
> SATA2 에서는 초당 300MB        
> SATA2 에서는 USB 처럼 허브를 이용해 하나의 포트에 여러 개의 하드를 연결할 수 있고, PC 를 끄지 않고 장치를 연결하는 핫플러그도 지원한다.          
> 최근에는 SATA3 도 나왔다. 초당 600MB      
* * *
**SCSI ( Small Computer System Interface )** 
> 서버나 워크스테이션 등에 쓰이는 고속 인터페이스다.      
> 안정성이 높은 것이 최대 장점          
> 가격이 비싼 것은 단점    
> 최신 규격인 울트라 320은 최대 320MB/s 의 속도를 낸다.
> 최근 SAS 가 등장하면서 조금씩 퇴색되고 있다.
* * *
**SAS (Serial Attached SCSI**
> SCSI 를 발전시킨 것으로 마찬가지로 서버나 워크스테이션 등에서 쓰인다.
> 울트라 320 SCSI 보다 좋다.
> 커넥터와 선은 SATA 와 같은 것을 사용한다.
SATA 규격 하드를 SAS 장치에 꽂아 쓸 수는 있지만, 반대로 SAS 하드를 SATA 장치에 꽂아 사용하지는 못한다.

Directory Sevice
--------

디렉터리 서비스는 네트워크 관련 자원 정보를 중앙의 저장소에 통합시켜 놓음으로써 더 이상 사용자가 네트워크상의 정보를 찾기 위해서 해매지 않도록 도와준다. 즉 사용자는 중앙의 저장소를
통해서 원하는 정보에 대한 정보를 자동으로 취득하게 되고 네트워크 자원에 접근할 수 있는것이다. 여기서 네트워크 자원이란 사용자계정,프린터 등의 주변기기를 말한다

Acitive Directory 도메인서비스
--------
컴퓨터,사용자 기타 주변장치에 대한 정보를 네트워크상 저장하면 Active Directory 도메인 서비스는 이러한 정보들을 관리자가 통합하여 관리하게 된다.
이렇게하면 사용자는 네트워크에 편리하게 자원들을 공유할수있으며 공동으로 작업도 가능해진다 Active Directory 도메인 서비스를 사용하기 위해서는 기본적으로
dns서버를 설치해야한다

DNS=DNS
-------
DNS는 도메인네임서버를 일컫는다. 인터넷은 서버들을 유일하게 구분할 수 있는 IP주소를 기본체계로 이용하는데 숫자로 이루어진 조합이라 인간이 기억하기에는 무리가 따른다. 따라서 DNS를 이용해 IP주소를 인간이 기억하기 편한 언어체계로 변환하는 작업이 필요한데 이 역할을 DNS가 하는 것이다.
그냥 요약하면 ip를 쓰기힘들니까 아이피를 언어올 바꾼거다 
도메인->www.naver.com 이런것들이 인간의언어로 되어있는이유는 dns서비스때문에 있는거다
그리고 www.naver.com 이거는 사실 www.naver.com. 뒤에 점이 하나 더 붙는다 이걸 루트동행이라한다 그래서 도메인 읽을떄는 뒤부터 읽어야한다 .(루트동행)(이건안보인다)->.com->naver->www. 이렇게 읽어야한다

순서
처음에는 .(루트도메인)시작하고
com=컴퍼니 학교는 ac=아카데미 청와대는 go = 거버먼트 mil=밀리터리 뒤에 끝나는 이런게 도메인의 첫번째 레벨 (1레벨)
naver, google 2번째 레벨이다 (2레벨)
www 3번째 레벨 (3레벨)

도메인
도메인은(Domain)은 Active Directory의 가장 기본이 되는 단위로써, 서울본사,부산지사 등이
각각 하나의 도메인이라고 보면 된다, 즉 도메인은 관리를 하기 위한 하나의 큰 단위의 범위를 표현하며,관리를 위해서 지역적인 범위로 구분할수있다
즉 도메인은 하나의 범위다

트리와 포리스트
---------
트리(Tree)는 도메인의 집함으로 보면된다. 부모 도메인인 서울지사와 자식 도메인인 부사지사와, 일본사무실이 계층적 구조로 되어 있는데 이를 하나로     
묶은 한빛 주식회사를 "트리 구조" 또는 그냥 "트리"라고 부른다 그래서 트리는 물리적으로 존재하다기보다는 개념적인 것으로 보면 된다. 한빛 주식회사는 단일 트리로 구성되어있는데   
만약 디스 주식회사라는 다른 회사와 전략적 제휴를 맺게 된다면 두 회사는 서로 관계를 맺을 수가 있을 것이다 , 즉 두 개 이상의트리로 Active Directory가 구성되는데 이를 포리스트라고한다

> 트리가 모이면 포레스트(포리스트)                      
> 도메인<트리<포리스트          


사이트
----------
도메인이 논리적인 범주라면, 사이트는(Site)는 물리적인 범주에 가깝다   
서울 본사와 일본 사무실은 Hambit.com으로 같은 도메인으로 묶여 있지만, 실제로는 거리상 상당히 떨어져 있다.    
그러므로 서울 본사와 일본 사무실은 같은 도메인이지만 별도의 사이트로 구성된다     


트러스트
---------
도메인 또는 포리스트 사이에 신뢰할 지 여부에 대한 관계를 나타내는 의미로 사용된다. 우선 같은
포리스트 안의 도메인 사이에는 상호 양방향전이 트러스틀(Trust)를 갖는다 이 외에도 한쪽 방향의 
신뢰관계만 맺은 경우를 '단반향 전이 트러스터'라 부른다



조직 구성 단위
------------
조직 구성 단위(organizational unit, OU)는 한 도메인 안에서 세부적인 단위로 나누는 것을 말한다.
서울 본사를 관리부,회계부,기술부 등의 부서로 나눈다면 이러한 부서를 '조직 구성 단위'라 함

글로벌 카탈로그
----------
Active Directory 트러스트 내의 도메인들에 포함된 개체에 대한 정보를 수집하여 저장되는 통합 저장소를
글로벌 카탈로그(Global Cartlog)라고 말한다. 예를 들어 사용자의 경우에는 사용자의 이름,전체 이름,로그인 아이디와 비밀번호 등의 정보가 글로벌 카탈로그에 저장된다.

원격 방법
======
1.원격 서비스 활성화
2.방화벽 열어둬야함
서버 관리자를 실행해서 왼쪽 로컬 서버를 선택하고 윈도우즈 방화벽에서 도메인:사용 또는 개인:사용을 클릭한다
고급설정 클릭
(팁 주자면 윈도우+r누르고 firewall.cpl입력하면  바로 고급설정가진다)
네트워크 엑세스 클릭하고 규칙사용 클릭
사진보고 3개 규칙사용클릭
first에서 원격 서버를 관리해보자
퍼스트에서 세컨드의 컴퓨터관리가 열린다 세컨드에 대한 모든 작업이 가능하다
이제부터는 서버관리자의 모든 서버에서 해당 컴퓨터를 선택하고
마우스 우클릭하면 대부분의 작업이 가능하다.
first에서 원격 데스크톱을 활용하자(윈도우키 누르고 보조메뉴에서 원격 데스크톱 클릭)
windows의 시작 > windows 보조 프로그램 >원격 데스크톱 연결을 선택하고 원격 데스크톱 연결 창에서
컴퓨터를 second.yonsai.com으로 연결한다

액티브 디렉터리 사용자 계정과 조직 구성 단위
===
사용자 계정은 Active Directory와 관련이 없이 로컬 컴퓨터만 접근할 수 있는 '로컬 사용자 계정'과
Active Directory 도메인에 접근할 수 있는 '도메인 사용자 계정'으로 나뉜다

'로컬 사용자 계정'은 Active Directory를 구성하기 이전에 사용하던 계정으로 이후로는 사용하지 않는다.
주로 언급하는 사용자 계정은 Active Directory를 그상한 이후의 사용자 계정인 도메인 사용자 계정이 된다
즉 로컬 컴퓨터뿐 아니라 Active Directory 도메인 전체에 대한 로그온 할 수 있는 사용자를 생성하고 관리할 것  이다

도메인 사용자 계정의 이름은 4가지 정도로 표현할 수 있는데, 만약 yonsai.com 도메인 (netbios)는
yonsai에 사용자 이름을 thisuser로 생성할 경우에 다음과 같이 나타낼 수 있다.

+ 기본적인 도메인 로그온 이름:YONSAI\thisuser 이거 시험에나옴 중요 ★
+ UPN : thisUser@yonsai.com
+ Dishguished name : CN=thisuser, OU=조직 구성단위이름 DC=yonsai, DC=com DC 2개는 도메인
+ Relative Dishguished name : CN:thisUser(조직 구성단위 안에서만 사용 가능)

+Dishguished name에서 CN은 Common Name, OU는 Organizational Unit, DC는
Domain Compoent를 의미한다 Organizational Unit(조직구성단위)
